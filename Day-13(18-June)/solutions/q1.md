# Q1 – Template-Driven Forms vs Reactive Forms

## Template-Driven Forms

- Form logic lives in the **HTML template** using directives like `ngModel`, `ngForm`, and `ngModelGroup`.
- Angular infers the form structure from the template.
- Less code in the component, but harder to unit test.
- Best for **simple forms** with straightforward validation needs.

## Reactive Forms

- Form logic lives in the **TypeScript component** using `FormGroup`, `FormControl`, and `FormBuilder`.
- Explicit and predictable — the developer defines every control and validator in code.
- Easier to unit test because the form model is just a plain object.
- Best for **complex forms** with dynamic fields, conditional validation, or custom validators.

---

## Real-World Examples

### When to choose Template-Driven Forms

1. **Newsletter subscription** – A single email field with a submit button. The form is trivial; putting the logic in the template keeps things simple and fast to build.
2. **Contact Us form** – A basic form with Name, Email, and Message fields and standard required/email validations. No dynamic behavior needed.

### When to choose Reactive Forms

1. **Job Application form** – Multiple sections (personal info, work history, references), conditional fields (show extra questions based on selected job role), and complex cross-field validation. All of this is much cleaner to manage in TypeScript.
2. **Multi-step checkout form** – Fields change based on user choices (e.g., shipping vs billing address toggle, payment method selection). Reactive Forms make it straightforward to add/remove controls programmatically and validate each step independently.
